Arquitetura

O design tem como objetivo obter uma aplicação com as seguintes características:

  • modularidade, para permitir a adição de novas funcionalidades
  • usabilidade, para que a experiência do usuário seja agradável e de fácil aprendizado
  • performance, para que o uso da aplicação não atrapalhe os usuários em seu trabalho

Componentes

Como se trata de uma aplicação web a ser disponibilizada na intranet, a estrutura básica da aplicação segue o modelo de cliente-servidor.

A persistência dos dados é essencial nesse sistema, e o modelo pode ser "estendido" para incluir uma terceira camada, resultando, então, numa estrutura com 3 componentes básicos: (1) banco de dados, (2) servidor web e (3) interface do usuário.

Essa maneira de pensar o sistema se reflete na organização do repositório projeto, o qual possui os seguintes diretórios:

  • database
  • backend
  • frontend
