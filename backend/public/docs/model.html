Modelo de Dados

A implementação do modelo de dados consiste em transformar as entidades, regras, restrições e requisitos contidos no modelo conceitural (conforme apresentado na visão geral) em objetos do banco de dados.

Os principais objetos utilizados nessa atividade são as tabelas.

Tabelas de referência

As tabelas de referência são listas taxativas de atributos válidos para classificar as entidades. Em geral, cada linha é formada por um código e um texto descritivo.

Como essas tabelas são referenciadas em restrições (do tipo chave-estrangeira) em colunas das tabelas que represenam as entidades, as tabelas de referência devem ser criadas antes de quaisquer outras.

Exemplos:

  • task_statuses, para os possíveis estágios de uma tarefa (pendente, em execução, concluída etc.)
  • spec_categories, para as possíveis classificações das especificações técnicas (civil, elétrica, marcenaria etc.)

Outras tabelas

As outras tabelas são criadas com o objetivo de traduzir as características de cada entidade do modelo conceitural, bem como os relacionamentos entre si e as funcionalidades desejadas para a aplicação.

Em alguns casos, é necessário o uso de tabelas de associação (em especial nos casos de relactionamentos do tipo 'muitos-para-muitos').

Exemplos:

  • tasks, que congrega as características essenciais de uma tarefa.
  • task_assets, que indica quais ativos estão vinculados a uma tarefa (tabela de associação)
